3.  custom_cat explanation:
To fulfill the requirements of Project 2 , I implemented a lightweight version of the cat utility, titled custom_cat, developed from scratch in C. 
The implementation primarily utilizes low-level Linux system calls, specifically open() to access the file, read() to pull data into a 4096-byte buffer, 
and write() to direct that data to the standard output (file descriptor 1). This approach demonstrates a fundamental understanding of kernel-user space interaction 
and manual buffer management. To execute the utility, I first compiled the source code using gcc and then ran the resulting binary through a custom-built shell program. 
Within the shell, the command is invoked by providing the path to the executable followed by the target filename (e.g., G5_Shell> ./custom_cat sample.txt), at which point 
the shell manages the process lifecycle through the fork(), execvp(), and wait() mechanism. This project highlights the core concepts of process creation, file descriptor 
manipulation, and the operational flow of a command-line interface within a UNIX-like environment.
